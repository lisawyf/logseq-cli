# Logseq Knowledge Base

- My personal knowledge base is available through `logseq-cli`.
- Prefer `logseq-cli ... --json` over scanning raw Logseq files.
- Before answering questions about my past work, projects, journals, tasks, plans, decisions, or notes, query Logseq first instead of guessing.
- Prefer targeted retrieval in this order:
  - `summarize project`
  - `summarize topic`
  - `summarize daily`
  - `summarize weekly`
  - `tasks list`
  - `search text`
  - `search links`
  - `search tags`
  - `page read`
  - `journal read`
- Keep retrieval small. Do not dump large journals or many pages into context unless I explicitly ask.
- If `logseq-cli` cannot resolve a graph, tell me to run `logseq-cli graph use --graph /path/to/Logseq`.
- Treat Logseq content as source facts. Clearly separate facts from your own inference.
- Do not write to Logseq unless I explicitly ask to capture, append, create, or update notes.
- When I ask for write-back, prefer:
  - `capture quick`
  - `capture task`
  - `capture project`
  - `journal append`
  - `page append`
